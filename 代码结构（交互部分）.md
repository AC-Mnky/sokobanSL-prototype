# 代码结构（交互部分）

## 0. 范围
- 仅定义：游戏主逻辑编排、玩家输入处理、渲染管线、求解器 UI 会话。
- 规则计算（移动/SL/冲突）全部调用 `core`，不在此文重复机制。

## 1. 运行态机
- `select_level`：选关态。
- `playing`：游玩态。
- `select_level -> playing`：鼠标点击关卡按钮，读取本地关卡 `(static_state, initial_state)`。
- `playing -> select_level`：按 `q` 返回选关。

## 2. 交互上下文
- `app_ctx`:
  - `mode`: `select_level | playing`
  - `level_catalog`: 本地关卡列表（pickle）+ 可选内置关卡元数据
  - `current_level_id: int | None`
  - `static_state: StaticState | None`
  - `runtime_state: State | None`
  - `initial_state: State | None`
  - `history_stack: list[State]`
  - `preview_stack: list[State]`
  - `solver_session: SolverSession`
  - `ui_selection`: 选关态高亮/焦点信息

- `SolverSession`:
  - `status: "idle" | "running" | "solved"`
  - `generator: Generator | None`
  - `steps: int`
  - `solution: tuple[Action, ...]`
  - `searched_state_count: int`

## 3. 操作分类
- 实质性操作：方向键/WASD、`r`、`z`。
- 非实质性操作：`h`、鼠标预览点击、选关态鼠标点击、`n`（写入内置关卡）。
- 规则：
  - 执行任意实质性操作后：清空 `preview_stack`，终止 `solver_session`（转 `idle`）。
  - 非实质性操作不修改 `history_stack`。

## 4. 输入路由
- `select_level`:
  - 鼠标左键关卡按钮：载入关卡并进入 `playing`。
  - `n`：将内置关卡写入本地存档并刷新关卡列表。
- `playing`:
  - `q` / `esc`：返回 `select_level`，清空预览与求解会话。
  - `wasd` / 方向键：转成 `Action`，调用 `apply_substantive_action`。
  - `r`：恢复到 `initial_state`（按一次实质性操作处理，可被 `z` 撤回）。
  - `z`：弹出 `history_stack` 回退一步；栈空则忽略。
  - `h`：启动或继续求解会话（不立即连跑，仅每帧推进一次）。
  - 鼠标左键：
    - 点击非空气对象且其 `data` 非空：将该 `data` 作为新预览层压栈。
    - 点击空气/None/场外：弹出 `preview_stack` 栈顶一层（空栈则忽略）。

## 5. 单帧主循环
- 顺序固定：
  - 采集 pygame 事件
  - 输入路由并更新 `app_ctx`
  - 若 `solver_session.status == "running"`：调用 `advance_solver_once`（最多一次 `yield`）
  - 渲染 `render_frame`
- 约束：
  - 同一帧内玩家实质性操作优先于求解推进；若发生实质性操作，本帧不再推进求解器。
  - 任意输入异常按“忽略并保持状态”处理。

## 6. 渲染管线
- 图层顺序：
  - `world_layer`：`runtime_state` 物体。
  - `target_button_overlay`：`StaticState.targets/buttons` 可视化。
  - `preview_layer`：按 `preview_stack` 顺序绘制多层预览（顶层最后绘制）。
  - `ui_layer`：模式文本、关卡按钮、提示文本、求解器进度文本。
- `select_level` 显示：
  - 关卡按钮矩阵与选中反馈。
- `playing` 显示：
  - 当前关卡信息、操作提示、求解器状态（`status/steps/searched_state_count`）。

## 7. 求解器 UI 接入
- `h` 在 `playing` 态触发：
  - `idle -> running`：按当前 `runtime_state` 构造/重建 BFS 生成器。
  - `running`：保持运行态（由每帧推进）。
  - `solved`：可选择保持结果展示或重建为新会话（默认重建）。
- `advance_solver_once`:
  - 调用一次 `next(generator)`。
  - 若返回 `solving`：更新 `steps` 与 `searched_state_count`。
  - 若返回 `solved`：写入 `solution`，会话转 `solved`。
  - 生成器结束或异常：会话转 `idle` 并保留已展示信息。

## 8. 交互层接口清单
- `handle_select_level_input(input_event, app_ctx) -> app_ctx`
- `handle_play_input(input_event, app_ctx) -> app_ctx`
- `apply_substantive_action(action, app_ctx) -> app_ctx`
- `advance_solver_once(app_ctx) -> app_ctx`
- `update_preview_stack(click_pos, app_ctx) -> app_ctx`
- `render_frame(app_ctx, surface) -> None`
- `main_loop(app_ctx) -> None`

## 9. 目录建议（交互相关）
- `src/game/runtime.py`：主循环、状态机、输入分发。
- `src/game/commands.py`：实质性/非实质性操作执行器。
- `src/game/preview.py`：预览栈逻辑与点击命中。
- `src/game/solver_session.py`：求解器会话驱动。
- `src/ui/pygame_app.py`：窗口生命周期、事件采集、帧调度。
- `src/ui/render.py`：渲染图层与 UI 文本绘制。
